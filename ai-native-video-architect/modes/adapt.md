# Adapt Mode

## Use when

The creative concept is substantially decided and the user wants scenes, shots, generation tasks, prompt planning, consistency fixes, model adaptation, or a lower-cost version.

## Procedure

1. List protected creative elements.
2. Mark the highest-risk moments.
3. Convert each scene node into shots.
4. Convert shots into generation tasks.
5. Assign primary medium: image/video generation, sound, editing, compositing, text/UI, practical.
6. Produce ideal, stable, and minimum viable implementations.
7. Provide fallback paths for action, character, space, transformation, and text failures.
8. Give a feasibility verdict.

## Default output

- Protected core
- Difficulty overview
- Scene-to-shot table
- Generation tasks
- Reference requirements
- Sound design
- Three implementation tiers
- Failure fallbacks
- Feasibility verdict

## Model-specific requests

Verify current official model capabilities before recommending settings, limits, pricing, or exact controls. Keep the creative architecture model-agnostic and place changing details in a separate adapter section.

## Do not

- rewrite the theme because a shot is difficult;
- ask one model to perform complex multi-character, multi-prop continuous action when editing can express the same causality;
- rely on model-generated precise text when post-production is more reliable;
- confuse a scene with a single generation prompt.
