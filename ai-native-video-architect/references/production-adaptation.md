# Production Adaptation

## Protect before simplifying

Record:

- unchangeable proposition;
- unchangeable primary mechanism;
- unchangeable trigger/tendency;
- unchangeable character choice;
- unchangeable ending meaning.

Simplify implementation, not these protected elements.

## Separate three units

### Scene node
A complete narrative event.

### Shot
A continuous view in the finished edit.

### Generation task
A model-generated asset segment. One shot may use several tasks.

Example:

- Scene: three doors open under wind pressure.
- Shot: wide view showing all doors now ajar.
- Tasks: left door state, right door state, rear door state, final composite, sound track.

## Risk matrix

Assess:

- character consistency;
- facial continuity;
- lip synchronization;
- hand/object interaction;
- multi-person contact;
- continuous action;
- space continuity;
- prop continuity;
- precise text/UI;
- multi-object movement;
- transformation stability;
- sound dependency;
- post-production workload.

## Three implementation tiers

### Ideal
Closest to the creative vision, even if technically demanding.

### Stable
Preferred default. Uses state changes, reference frames, separate reaction shots, sound, and compositing.

### Minimum viable
The cheapest reliable expression that preserves meaning.

Examples:

- continuous fall → dropped object sound + result shot;
- simultaneous doors → separate openings + final wide state;
- moving chair → wheel sound + reaction + chair in new position;
- complex transformation → first/last frame + cutaway + changed result;
- accurate Chinese UI → post-production overlay rather than model-generated text.

## Reference package

List required:

- character references;
- clothing references;
- space layout;
- prop references;
- lighting anchors;
- first/last frames;
- sound assets;
- exact text overlays;
- continuity notes.

## Sound is part of the rule

For every key scene identify:

- sound trigger;
- sound response;
- sound escalation;
- use of silence;
- synchronization or mismatch with image.

## Feasibility verdicts

- directly producible;
- producible with conditions;
- requires shot restructuring;
- not recommended under current constraints.

Never base feasibility only on whether a single attractive shot can be generated.
