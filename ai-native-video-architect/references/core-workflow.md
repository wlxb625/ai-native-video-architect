# Core Workflow

Use this workflow internally for every mode. Show only the portions useful to the user.

## 1. Parse the project

Extract or infer:

- source material: topic, emotion, plot, script, music, product, or existing plan;
- desired format and duration;
- main and secondary genre;
- audience and viewing context;
- realism level and narrative density;
- fixed characters, ending, product facts, or visual style;
- production constraints and named models.

Use defaults instead of interrogating the user unless a missing answer changes the creative direction materially.

## 2. Diagnose the visible and invisible layers

Distinguish:

1. **Surface topic** — workplace, family, school, grief, navigation.
2. **Concrete situation** — what a specific character is doing and facing.
3. **Invisible conflict** — what important relationship is not visually apparent.
4. **Core proposition** — a relational statement, not a theme word.

A strong proposition includes two forces and a consequence.

Example:

> The more a worker makes themselves continuously available to prove commitment, the more the organization treats availability—not contribution—as their value.

## 3. Generate mechanism candidates

Generate 3–5 candidates internally. Make them structurally different, not cosmetic variants.

Across candidates vary at least three of:

- abnormal source;
- primary carrier;
- rule hardness;
- benefit structure;
- symmetry;
- character relation;
- escalation dimension;
- ending payoff.

Candidate card:

- name;
- proposition link;
- abnormal source;
- carrier;
- one-line rule/tendency;
- effect on action;
- escalation;
- genre fit;
- production risk;
- cliché risk.

## 4. Select the primary mechanism

Select based on:

- thematic specificity;
- narrative participation;
- genre fit;
- visual/auditory evolution;
- medium non-substitutability;
- originality without obscurity;
- production adaptability.

Default to one primary mechanism and at most one supporting mechanism.

State the irreplaceable reason:

> Why does this particular source and carrier express this situation more accurately than generic disappearance, transparency, distortion, or repetition?

## 5. Build the world behavior

Choose rule hardness:

- **Hard rule:** precise trigger and response.
- **Medium rule:** stable pattern with limited uncertainty.
- **Soft rule:** coherent tendency without full predictability.
- **Ruleless but structured anomaly:** no stable operational law, but thematic and sensory direction remains consistent.

Hard-rule card:

- trigger;
- first response;
- repetition pattern;
- escalation dimensions;
- benefit structure;
- symmetry;
- character interpretation;
- use/resistance;
- irreversible threshold;
- closure.

Soft-rule card:

- abnormal tendency;
- conditions often associated with it;
- stable sensory direction;
- incorrect patterns characters may infer;
- boundaries that must remain unknowable;
- ending transformation.

## 6. Apply genre control

Load the relevant controller. Decide:

- mechanism visibility;
- rule explanation timing;
- audience prediction relationship;
- reality-anchor density;
- harm threshold;
- escalation shape;
- character–rule relationship;
- ending family;
- audience rule-knowledge ratio.

## 7. Build linked narrative lines

At minimum track:

- **Reality event line:** what characters do in the ordinary world.
- **World-change line:** how the mechanism evolves.
- **Character understanding line:** how interpretation changes.

Causal target:

`real action → world response → changed situation → new choice → escalated/altered rule`

Reject parallel decorative structures in which the plot proceeds independently while strange imagery happens in the background.

## 8. Build the scene graph

Each scene node should contain:

- reality event;
- immediate character goal;
- trigger/tendency;
- visible or audible change;
- character interpretation;
- resulting choice;
- visual escalation;
- plot escalation;
- thematic escalation;
- genre function;
- audience knowledge;
- sound behavior;
- production medium;
- what is lost if the scene is deleted.

Delete or merge nodes that repeat only the visual effect.

## 9. Run internal pressure tests

- **Remove-AI test:** what collapses if the mechanism is removed?
- **Replacement test:** can a generic effect replace it without changing meaning?
- **Causal test:** would the character choose the same action without the mechanism?
- **Random spectacle test:** can every major anomaly explain why it occurs and what it changes?
- **Genre consistency test:** is genre expressed through narrative presentation, not just visuals?
- **Reality responsibility test:** for social realism, who causes and can change the real problem?
- **Production test:** is there a stable fallback expression for the hardest moment?

## 10. Output by mode and depth

Do not expose the full workflow unless requested. Use the mode template and user-selected or inferred depth.
