# Character Agency Check

## 职责
检查人物或主体通过行动、认知、关系、边界、不行动、见证或形式偏离参与变化。

## 统一输出
```yaml
status: PASS | CONDITIONAL | FAIL
design_tags: []
applicability: HIGH | MEDIUM | LOW | NOT_APPLICABLE
evidence: []
risks: []
must_fix: []
should_strengthen: []
must_protect: []
```

## 评估维度
- 当前目标或状态
- 限制与资源
- 第一次行动及后果
- 中段策略/理解变化
- 高潮选择与代价
- 有限与分布式能动性
- 系统/配角/巧合替代风险

## 合法设计标签
- `ACTION_AGENCY`
- `COGNITIVE_AGENCY`
- `RELATIONAL_AGENCY`
- `BOUNDARY_AGENCY`
- `ACCEPTANCE_AGENCY`
- `NON_ACTION_AS_CHOICE`
- `INTENTIONAL_PASSIVITY`
- `LIMITED_AGENCY`
- `DISTRIBUTED_AGENCY`
- `STRUCTURAL_POWERLESSNESS`

## 硬失败
- 1. 人物无不可替代功能
- 2. 行动长期无后果
- 3. 高潮完全由临时他人解决
- 4. 系统替代人物全部选择
- 5. 被动没有限制/变化/后果
- 6. 改写无授权英雄化人物

## 条件通过
- 1. 人物成立但第一次行动略晚
- 2. 有限选择不够可见
- 3. 群像职责略重叠
- 4. 高潮选择清楚但代价偏弱
