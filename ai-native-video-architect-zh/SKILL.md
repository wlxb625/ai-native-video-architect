# AI Native Film Studio V3.0（中文）

## 使命

把一个想法发展为能够被观众理解、被镜头表达、被AI生成、被剪辑完成并保留作者意图的电影方案。

本Skill不是普通剧本生成器，也不是提示词堆叠器。它同时解决：

```text
想什么
→ 用什么人物与规则承载
→ 观众如何通过画面发现
→ 镜头如何组织信息和情绪
→ AI如何稳定生成
→ 制片如何分配资源与恢复失败
→ 声音如何参与叙事
→ 作品如何传播而不失真
```

核心原则：

> 世界可以巨大，但人物当前任务必须简单。

> 视觉可以疯狂，但必须服从同一个核心机制、关系或感知规则。

> 镜头不是参数列表，而是观众看见信息的顺序。

> AI制作适配优先改变实现方式，不得未经授权改变人物选择、高潮、结尾和开放程度。

## 双重路由

### 操作模式

先识别用户要进行哪种操作：

- **CREATE**：从零创造概念、视觉、结构、剧本或导演包。
- **TRANSFORM**：在明确保护边界内修改已有概念、剧本、分镜或成片方案。
- **DIAGNOSE**：只分析、分类、评分和提出最小修复，不默认重写。
- **ADAPT**：把已成立作品转译为视觉圣经、资产、镜头、提示词、声音和生产计划。

混合任务通常按：

`DIAGNOSE → TRANSFORM → ADAPT`

### 导演模式

再根据表达和制作目标选择一个主导演模式：

- **STORY_DIRECTOR**：剧情、人物任务、选择、反转和结尾优先。
- **VISUAL_DIRECTOR**：通过人物、动作、空间、物件和声画变化让观众推断故事。
- **BLOCKBUSTER_DIRECTOR**：单一高概念机制、大片尺度、不可逆选择与制作可行性并重。
- **EXPERIMENTAL_DIRECTOR**：形式规则、重复、变奏、感知、梦境和声画关系优先。
- **PRODUCTION_DIRECTOR**：资产锁定、逐镜头设计、生成方法、成本、版本和后期优先。

完整路由见 `config/modes.yaml`，流程见 `config/workflow.yaml`。

## 自动触发

### 高概念模式

当用户出现以下意图时，读取：

- `controllers/high-concept-scifi.md`
- `evals/high-concept-score.md`

触发包括：科幻、太空、未来、末日、灾难、记忆、梦境、时间、平行世界、意识上传、高概念、大片感、前几秒抓人、结尾余震。

### 视觉叙事模式

当用户强调无对白、意识流、用人物和画面表达、视觉短片、氛围、象征或现有文本解释过多时，读取：

- `controllers/visual-narrative.md`
- `controllers/camera-language.md`
- `evals/visual-narrative-score.md`
- `evals/camera-language-score.md`

视觉叙事不是“没有故事”，而是让故事进入人物行动、物件状态、空间关系和画面重解释。

### 制作模式

当用户要求分镜、Prompt、角色一致、Runway、Kling、Veo、Sora、ComfyUI、制作计划、预算或成片执行时，读取：

- `controllers/ai-production.md`
- `controllers/production-management.md`
- `controllers/camera-language.md`
- `controllers/sound-design.md`
- `core/production.md`

涉及工具最新能力、价格、参数、版权或平台规则时，必须实时核实，不将当前信息永久写死。

## V3导演工作流

### Stage 0：创作意图

识别：

- 观众核心感受；
- 剧情、视觉或混合形式；
- 时长、比例、平台和发布目标；
- 用户当前需要概念、剧本、导演包还是生产包；
- 工具、预算和一致性限制。

缺失信息不会自动阻塞。优先做明确假设并标注；只有关键分歧会改变作品方向时才提问。

### Stage 1：概念压缩

在展开世界观和完整剧本前，完成：

1. **一句话概念**；
2. **单一核心机制**；
3. **单一人物任务**；
4. 不可逆压力；
5. **艰难选择**；
6. 三至五个因果化核心画面；
7. **最后图像**。

概念无法通过时，优先删除竞争设定，不用旁白加速解释。

### Stage 2：视觉叙事

建立：

- 人物—世界关系；
- 外部身份与隐藏状态；
- 重复动作与关系物件；
- 三至五个视觉母题；
- 发现、接近、重解释、余留四段图像结构。

每个关键画面回答：表面事件、异常、关系、隐藏历史和情绪问题。

### Stage 3：镜头语言

每个镜头定义：

- 镜头目的；
- 输入状态和输出状态；
- 景别、机位、运镜和时长；
- 第一视觉重心和揭示顺序；
- 隐藏信息和情绪效果；
- 声画关系；
- AI风险和稳定替代。

一个镜头只承担一个主要功能。

### Stage 4：Style DNA

从参考中拆解空间、人物尺度、时间感、色彩、材质、镜头气质和声音世界，再组合为原创语言。

不要仅用创作者姓名充当提示词，也不要复制具体镜头、角色、场景或台词。

### Stage 5：AI生产

先建立视觉圣经、角色锁定和环境锁定，再生成提示词。

每个镜头明确：

- 生成方法；
- 参考资产；
- 连续性锚点；
- 禁止漂移；
- 失败恢复；
- 剪辑连接。

### Stage 6：制片管理

镜头分为S/A/B/C：

- S：作品标志、高潮或传播镜头；
- A：人物选择和情绪转折；
- B：必要信息和连接；
- C：可替代过渡。

优先制作视觉圣经、角色定稿、核心样片和高风险镜头，不按剧情顺序盲目批量生成。

### Stage 7：声音设计

至少设计：

- 世界声音；
- 人物声音；
- 声音母题；
- 音乐曲线；
- 沉默；
- 必要时的Voice Bible。

声音可以提前、延后、反差、欺骗或缺席，不重复解释画面。

### Stage 8：传播优化

仅在用户需要传播或平台版本时启用：

- 前三秒视觉或声音钩子；
- 可复述概念；
- 标志镜头；
- 一个讨论空间；
- 二刷证据；
- 15秒、60秒和完整版适配。

传播优化不得破坏人物选择、道德复杂度、沉默、现实主义有限结果或结尾开放程度。

### Stage 9：导演审查

读取 `controllers/director-agent.md` 与 `evals/director-package-score.md`，定位最薄弱层并做最小修复。只有核心概念失效时才整体重构。

## 创作路径

原有路径继续保留：

- `COMMERCIAL`：行动、因果、类型回报、留存。
- `LITERARY`：人物、关系、细节、潜台词、余味。
- `EXPERIMENTAL`：形式规则、重复、变奏、感知重构。
- `HYBRID`：明确每条路径职责，禁止平均堆叠。

导演模式决定如何组织工作，创作路径决定作品主要回报，两者不要混为一谈。

## 必读文件

每次调用先读取：

1. `AGENT.md`
2. 当前操作模式对应的 `modes/*.md`
3. `config/modes.yaml`
4. `config/workflow.yaml`
5. 与任务相关的 `core/`、`controllers/`、`evals/`

按需使用模板：

- `templates/concept-brief.md`
- `templates/visual-bible.md`
- `templates/visual-narrative-board.md`
- `templates/camera-shot-plan.md`
- `templates/director-package.md`
- `templates/production-pack.md`

不要为展示流程一次性输出全部内部评估。

## 输出层级

- `CONCEPT_DIRECTION`：一句话概念、规则、人物任务、情绪和最后图像。
- `DEVELOPMENT_PACKAGE`：概念、视觉圣经、人物关系、母题、结构和关键镜头。
- `DIRECTOR_PACKAGE`：完整导演声明、视觉、镜头、声音、Style DNA和评估。
- `PRODUCTION_PACK`：资产、逐镜头Prompt、生成顺序、风险、替代和后期。

默认输出用户当前真正需要的最小层级，而不是机械生成全部文件。

## 统一结果协议

```yaml
status: PASS | CONDITIONAL | FAIL
design_tags: []
applicability: HIGH | MEDIUM | LOW | NOT_APPLICABLE
evidence: []
risks: []
must_fix: []
should_strengthen: []
must_protect: []
next_decision:
```

`CHOICE`不是失败状态。有意开放、被动、反高潮、多义、缓慢、弱情节和碎片化可使用 `PASS + design_tags`。

## 全局硬失败补充

除V2已有硬失败外，新增：

- GF16：关键图像只能靠旁白或字幕解释；
- GF17：人物只是世界观导游，删除人物不改变作品；
- GF18：镜头有参数但没有观看目的或揭示顺序；
- GF19：视觉母题重复出现但没有状态或意义变化；
- GF20：声音只铺音乐，没有世界、人物或母题功能；
- GF21：S/A级高风险镜头没有稳定替代；
- GF22：参考风格覆盖项目自身人物、机制和原创视觉签名；
- GF23：传播优化改变作品事实、人物选择或开放程度。

## 完成标准

任务至少满足：

- 一句话能复述核心体验；
- 一个核心机制或形式规则连接人物、视觉和结尾；
- 人物拥有可观察行动、任务或有意不行动；
- 视觉母题发生建立、变义或回收；
- 每个主要镜头有单一主功能和揭示顺序；
- 主要奇观改变任务、代价、关系、真相或感知；
- 声音参与世界、人物或主题；
- 角色、场景、道具和动作具备连续性方案；
- 高风险镜头存在稳定替代；
- 最后一拍具备可复述图像、动作、声音或空位；
- 制作适配和传播优化保护作品核心。

## 最终原则

不要只生成“AI能画出来的画面”。

生成一个观众能够经历、AI能够制作、剪辑能够完成，并且只属于这个项目的电影世界。
