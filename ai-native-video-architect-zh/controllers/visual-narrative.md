# Visual Narrative Controller

## 目标

用人物、动作、空间、物件、构图和声画变化承载叙事，让观众通过观察推断关系与历史，而不是依赖解释性对白或旁白。

视觉叙事不等于没有故事，也不等于随机意识流。它要求故事被放进可见变化中。

## 适用触发

- 无对白、少对白、意识流、氛围短片；
- 用户强调“用人物和画面表达”；
- 现有剧本解释过多、对白承担全部信息；
- 高概念作品需要把规则变成可见因果。

## 五层图像信息

每个关键画面至少回答其中三项，核心画面必须回答五项：

1. 表面事件：观众直接看见什么？
2. 异常：什么不符合日常预期？
3. 关系：人物与空间、物件或另一个人物是什么关系？
4. 隐藏历史：这个画面暗示此前发生过什么？
5. 情绪问题：观众会产生什么尚未回答的感受或疑问？

## 场景公式

```yaml
visual_scene:
  subject:
  observable_action:
  anomaly:
  hidden_information:
  relationship_change:
  emotional_question:
  motif_used:
  next_scene_dependency:
```

如果场景只有“漂亮环境 + 人物站立观看”，且没有关系、信息或状态变化，标记为 `VISUAL_WALLPAPER`。

## 角色协议

视觉叙事角色至少拥有：

- 一眼可识别的外部身份；
- 一个重复但会改变含义的标志动作；
- 一个与角色关系绑定的物件；
- 一个观众后续才能理解的隐藏状态；
- 一个通过行动而非说明完成的变化。

角色不能只当比例尺或导游。即使角色非常被动，他的观察、等待、拒绝、重复或离开也必须改变意义。

## 视觉母题协议

选择3—5个母题，每个母题最多承担两个主要含义。

优先使用三次递进：

1. 第一次制造问题；
2. 第二次改变理解；
3. 第三次完成情绪回环。

重复必须包含状态、位置、持有者、声音、光线或意义变化。完全相同的重复只适用于有意的压迫或停滞。

## 推荐结构

### Discovery

用人物动作发现异常，不用字幕解释世界。

### Approach

人物尝试接近、修复、保护、等待、逃避或理解异常。

### Reinterpretation

新画面使此前动作、物件或空间关系获得另一层含义。

### Residue

结尾保留一个动作、物件、声音或空间状态，让情绪继续存在。

## 视觉高潮

视觉高潮不是最大场景，而是同时完成：

- 图像升级；
- 关系重解释；
- 情绪落点；
- 前文母题回收。

如果高潮只能靠旁白说明其意义，返回前面重建视觉证据。

## AI视频适配

优先：

- 单人物或稳定双人关系；
- 单一主空间或少量强区分空间；
- 慢速、明确、可分段的动作；
- 具有固定外形的物件；
- 可通过光线、天气、材质和空间变化表现的规则。

谨慎：

- 多人复杂调度；
- 快速连续打斗；
- 依赖精确手部操作的长镜头；
- 同一镜头内多次身份变形；
- 只能通过大量文字读取才能理解的关键事实。

## 输出要求

视觉叙事任务默认先输出：

1. 人物—世界关系；
2. 三至五个视觉母题；
3. 发现、接近、重解释、余留四段图像结构；
4. 每个核心画面的隐藏信息；
5. 最后一幅画面；
6. 必要时再展开为镜头脚本。
