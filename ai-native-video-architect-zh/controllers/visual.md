# Visual Controller

视觉承担空间、关系、信息、意象和形式。它负责视觉系统总览；涉及人物与画面推断时转入 `controllers/visual-narrative.md`，涉及观看顺序与运镜时转入 `controllers/camera-language.md`，涉及锁定和生成时转入 `controllers/ai-production.md`。

## 视觉因果

高概念与意识流作品中的主要视觉场面必须来自核心机制、人物关系或稳定形式规则，而不是独立概念图。

```yaml
visual_event:
  image:
  source_rule:
  why_now:
  character_world_relationship:
  story_or_meaning_change:
  task_change:
  cost_or_reveal:
  motif_progression:
  setup_for_final_image:
```

若删除该画面后，任务、危险、代价、关系、真相、感知和结尾均不改变，则标记：

`DECORATIVE_SPECTACLE` 或 `VISUAL_WALLPAPER`

处理顺序：删除 → 与其他场面合并 → 赋予关系、信息或形式后果。

## 五层视觉信息

核心画面检查：

1. 表面看见什么；
2. 异常在哪里；
3. 人物与世界、物件或他人的关系；
4. 暗示了什么隐藏历史；
5. 留下什么情绪问题。

不是每个镜头都需要五层，但作品的发现、重解释和结尾镜头必须形成递进。

## 视觉母机制

一部高概念短片优先只设一条视觉母机制，例如：

- 被遗忘的事物失去重力；
- 记忆被燃烧时，城市被拉成推进光流；
- 谎言会改变空间几何；
- 时间债务让未来场景提前侵入现在；
- 被删除的人先失去声音，再失去五官，最后只剩动作。

所有变奏必须让观众逐渐理解同一规则，不应每场引入新超现实语法。

实验作品可以使用感知或形式母机制，例如每次门打开，人物年龄不变但房间生活痕迹向前十年。

## 视觉母题

选择3—5个可稳定识别的物件、动作、颜色、声音、材质或构图。

母题优先三次递进：

1. 建立问题；
2. 改变理解；
3. 完成情绪回收。

完全相同的重复只适用于有意压迫、停滞或仪式。其他情况必须改变位置、持有者、状态、声音或意义。

## 核心镜头预算

大片感默认集中在3—5个 `HERO_SHOT`：

1. `HOOK_SHOT`：建立异常；
2. `RULE_SHOT`：完整展示核心机制；
3. `ESCALATION_SHOT`：证明规模、关系或代价升级；
4. `CLIMAX_SHOT`：人物行动产生不可逆后果；
5. `AFTERIMAGE_SHOT`：留下可复述残余。

视觉叙事项目也可以将HERO_SHOT定义为微小动作或空位，不要求巨大尺度。

## 标志物

标志物至少承担两项：

- 人物关系；
- 核心机制锚点；
- 转场连接；
- 高潮触发；
- 结尾残余；
- 声音母题来源。

只承担美术统一时，不算叙事标志物。

## 意识流转场

转场优先通过共同形状、动作、声音、材质、视线或规则完成，例如：

- 海浪折叠成床单；
- 飞船舱门变成儿童房门；
- 星环与唱片旋转匹配；
- 呼吸声变成引擎点火；
- 手中物件改变重力并带入下一空间。

转场必须说明意识为何在此跳跃，以及跳跃改变了什么。

## 视觉与镜头边界

Visual Controller决定：

- 什么值得被看见；
- 哪些元素稳定或变化；
- 图像如何承载关系与母题。

Camera Language Controller决定：

- 观众先看哪里；
- 信息何时揭示；
- 景别、机位、运镜和停留如何产生情绪。

不要用“大景别、慢推进、电影感”等摄影词替代视觉内容设计。

## 制作保护

- 先锁定视觉圣经、角色、场景和核心镜头；
- 复杂人群优先剪影、光点、远景与重复资产；
- 复杂空间变形优先匹配剪辑、遮罩、粒子和分层合成；
- 文字、界面和精确符号优先后期制作；
- 降级时保留规则、关系、人物行动和最后图像，先减少规模；
- S/A镜头必须有稳定替代。
