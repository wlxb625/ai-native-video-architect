# Camera Language Controller

## 目标

把“画面里有什么”转化为“观众如何看见、何时理解、为什么产生情绪”。镜头语言不是装饰参数，而是信息与关系的组织方式。

## 六层镜头定义

每个正式镜头至少定义：

1. 景别：观众与人物或世界的距离；
2. 机位：力量、脆弱、控制或失衡关系；
3. 运镜：接近秘密、揭示孤独、跟随行动或保持审视；
4. 时长：信息被理解和情绪形成所需时间；
5. 视觉优先级：第一眼、第二眼和最后发现的位置；
6. 揭示顺序：观众先知道什么、后知道什么、最终重解释什么。

## 单镜头主功能

一个镜头只能有一个主要功能：

- 建立空间；
- 发现异常；
- 观察行动；
- 揭示关系；
- 改变理解；
- 承担情绪停留；
- 完成转场或连接。

可以存在辅助功能，但不能让一个镜头同时承担世界说明、人物背景、反转、动作高潮和主题总结。

## 运镜动机

- 推进：接近秘密、选择或人物内部状态；
- 拉远：揭示人物规模、失去、隔离或世界代价；
- 横移：发现空间关系、并置或被隐藏的对象；
- 环绕：关系不稳定、认知变化或仪式感；
- 跟随：强调任务、阻力和身体行动；
- 静止：让动作、环境或沉默承担意义；
- 手持或不稳定：仅在人物感知失衡时使用，不能作为廉价紧张感。

运镜没有情绪或信息原因时，改为静止或更简单的运动。

## 常用揭示公式

### 异常递进

异常细节 → 扩大空间 → 人物进入关系。

### 动作关系

人物动作 → 物件反馈 → 暗示关系或历史。

### A变B

先让观众按A理解画面 → 通过位置、反射、声音或拉远揭示B。

### 余韵停留

动作完成 → 不立即切走 → 让环境、声音或空位继续表达。

## 声画协同

镜头设计必须说明声音是：

- 同步支持；
- 提前预告；
- 延后残留；
- 反向制造冲突；
- 主动消失形成沉默。

重要揭示不应同时被画面、对白、字幕和音乐重复解释。

## AI视频可生成性

优先：

- 静止机位中的环境变化；
- 缓慢推进或横移；
- 单一清晰动作；
- 前景、中景、背景层次明确；
- 4—8秒内完成一个动作或一次揭示；
- 前后镜头存在清楚的姿态、视线、方向或物件连接。

谨慎：

- 高速自由穿越复杂空间；
- 多人物交叉走位；
- 长时间精确口型和表演；
- 同一镜头连续完成多个复杂事件；
- 纯靠“电影感”“史诗感”而没有具体运动逻辑的提示词。

## 详细分镜模式

当用户要求具体分镜、逐镜头提示词、电影级生成描述或可以直接投入AI视频制作的镜头表时，必须读取：

- `controllers/detailed-storyboard.md`；
- `templates/detailed-storyboard.md`。

详细分镜必须先给出全片摄影基准，再逐镜头展开。用户未指定时，默认采用：

- 21:9横向宽银幕；
- ARRI Alexa 35或ARRI Alexa LF成像参考；
- 变形宽银幕镜头特征；
- 24fps；
- 180度快门；
- 真实演员和真实材质；
- 柔和高光滚降；
- 暗部保留层次；
- 固定、极慢推进、缓慢后拉、稳定横移、稳定跟拍或低幅度焦点转移。

不得用“电影感、史诗感、高级感”代替具体构图、光线、人物动作、材质、运镜和连续性描述。

每个详细分镜至少补充：

- 21:9中主体的准确位置和占比；
- 前景、中景、背景；
- 摄影机和变形宽银幕光学基准；
- 焦段或焦段感；
- 运镜起止位置、速度和动机；
- 人物微表演和动作全过程；
- 光源方向、色温、曝光关系、高光和暗部；
- 服装、建筑、家具和道具的真实使用痕迹；
- 帧率、快门和运动模糊；
- 相邻镜头连续性；
- AI风险、稳定替代、中文Prompt、英文Prompt和负面约束。

## 高潮与结尾

高潮必须至少完成两项：

- 视觉尺度升级；
- 关系重新解释；
- 人物做出不可逆行动；
- 声画规则发生有准备的变化。

结尾优先使用：

- 物件留下、人物离开；
- 重复动作获得新含义；
- 先前图像被反向理解；
- 声音继续而画面停止；
- 空间恢复平静但关系已改变。
