# 场景图谱设计

场景图谱位于世界规则与完整剧本之间，用于检查结构，不是镜头表。

## 基础字段

| 字段 | 说明 |
|---|---|
| 场景编号 | 顺序 |
| 时间与空间 | 现实锚点 |
| 现实事件 | 发生了什么 |
| 人物目标 | 当前想完成什么 |
| 规则触发/异常倾向 | 为什么出现变化 |
| 可见或可听变化 | 观众具体感知什么 |
| 人物解释 | 人物怎样理解 |
| 人物选择 | 本场后做什么 |
| 视觉/听觉升级 | 表现层新变化 |
| 情节升级 | 风险、目标或关系变化 |
| 人物认知升级 | 理解进入什么层次 |
| 主题升级 | 意义进入什么更深层 |
| 类型功能 | 共情、恐惧、悬疑或笑点 |
| 观众知晓度 | 规则被理解多少 |
| 主要媒介 | 画面、声音、文字、剪辑或合成 |
| 场景结果 | 删除本场会失去什么 |

## 类型附加字段

### 现实主义

- 现实责任；
- 类型漂移风险。

### 恐怖

- 人物的错误解释；
- 本场保留的未知。

### 喜剧

- 笑点升级；
- 伤害等级。

## 节点数量建议

- 30—60 秒：3—5 个；
- 60—120 秒：5—8 个；
- 3—5 分钟：8—15 个。

镜头数量可以更多，但结构节点不宜过度增加。

## 节点功能

常见五阶段：

1. 建立人物与现实；
2. 第一次异常；
3. 重复并确认规律/倾向；
4. 规则改变目标或人物作出关键选择；
5. 结尾兑现或继续。

这不是固定模板。恐怖可以使用规则冲突或软侵蚀，喜剧可以成功利用或形成新秩序，现实主义可以以认知变化和未解决现实收束。

## 删除测试

删除某场后，如果规则、人物认知、事件和结尾仍完整，该场可能只是重复展示，应合并或删除。
