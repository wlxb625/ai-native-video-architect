# AI 原生视频核心工作流

## 目标

把题材、情绪、人物处境或已有剧本，转化为由生成机制参与意义与叙事的可执行视频方案。

完整流程：

`输入解析 → 内核诊断 → 机制发散 → 机制筛选 → 世界规则 → 类型控制 → 双线结构 → 场景图谱 → 压力测试 → 剧本/制作转换`

## 一、输入解析

提取：

- 视频类型、时长、平台、受众；
- 剧情密度、对白密度、现实程度；
- 用户希望观众感受和理解什么；
- 人物一致性、场景数量、口型、指定模型等制作限制。

信息不足时采用 SKILL.md 默认值。只有缺失信息会改变作品本质时才询问。

## 二、提炼不可见内核

区分四层：

1. 表面题材；
2. 人物具体处境；
3. 不可见矛盾；
4. 核心命题。

核心命题必须是关系判断，而不是单个词。

例如：

- 不合格：孤独。
- 合格：人物越努力持续在线证明自己参与群体，群体越只用在线时间判断他的价值。

依次回答：

1. 人物表面上在做什么？
2. 真正想得到什么？
3. 为什么得不到？
4. 哪种冲突在现实中难以直接看见？
5. 作品希望改变观众对什么事情的理解？

## 三、机制发散

至少内部生成 3 个候选，通常不超过 5 个。

候选必须在以下方面存在结构差异：

- 异常来源；
- 主要承载体；
- 规则利益结构；
- 人物与规则关系；
- 类型潜力；
- 结尾可能性。

每个候选包含：

- 名称；
- 对应命题；
- 异常来源；
- 承载体；
- 一句话规则；
- 人物如何受影响；
- 升级方向；
- 风险。

候选不能只是“透明、半透明、逐渐消失”的表面变化。

## 四、机制筛选

建议内部评分：

- 主题具体相关性 25；
- 叙事推动力 20；
- 类型适配 15；
- 视觉/听觉演化潜力 15；
- 媒介不可替代性 10；
- 新颖度 5；
- 生产可行性 5；
- 兼容性 5。

默认采用：

`一个主机制 + 最多一个辅助机制 + 一个稳定母题`

主机制需要能写出至少三个阶段的变化，并说明不可替代原因。

## 五、世界规则

硬规则回答：

- 什么触发？
- 世界如何响应？
- 第二次与第一次有什么不同？
- 如何升级？
- 对谁有利、对谁有害？
- 人物如何理解、利用、抵抗或协商？
- 不可逆节点是什么？
- 如何终止、继续或改变意义？

软规则记录：

- 异常倾向；
- 经常出现的条件；
- 已知但不稳定的关联；
- 人物的错误总结；
- 不能被验证的边界；
- 必须保留的未知。

## 六、类型控制

类型控制卡包含：

- 主类型、次类型；
- AI 介入强度；
- 机制可见度；
- 现实锚点密度；
- 规则解释策略；
- 观众预测关系；
- 主要升级维度；
- 人物与规则关系；
- 伤害阈值；
- 结尾形式；
- 现实责任或未知保存要求。

类型不能只靠色调和美术风格区分。

## 七、建立双线结构

### 现实事件线

人物做了什么、想完成什么、现实阻力是什么、作出什么选择。

### 世界变化线

规则如何被触发、如何显现、如何升级、人物如何重新解释。

合格的双向因果：

`现实行为触发变化 → 变化改变处境 → 人物作出新选择 → 新选择进一步改变规则或现实事件`

世界变化不能只是背景氛围。

## 八、场景图谱

每个场景至少记录：

- 现实事件；
- 人物目标；
- 规则触发或异常倾向；
- 可见/可听变化；
- 人物解释与选择；
- 视觉、情节、认知和主题升级；
- 类型功能；
- 观众知晓度；
- 场景结果；
- 主要制作媒介。

删除本场后，如果人物、规则、观众理解和结尾仍完整，该场可能只是重复展示。

## 九、压力测试

### 去 AI 测试

删除生成异常后，作品核心意义和情节是否明显受损？

### 机制替换测试

把机制换成透明、镜子、粒子等常见效果，意义是否几乎不变？若能随便替换，主题耦合不足。

### 因果参与测试

没有世界变化时，人物是否仍会作出相同选择？如果始终相同，机制没有进入剧情。

### 随机奇观测试

每个主要异常必须回答：为什么发生、由什么触发、表达什么、如何呼应、如何影响下一场。

### 类型一致性测试

检查规则解释、可见度、节奏、伤害和结尾是否符合类型。

## 十、剧本转换

每场可使用六层结构：

- 现实层；
- 生成层；
- 人物层；
- 类型层；
- 母题层；
- 声音层。

对白应推动现实关系、暴露误解或触发规则，避免重复解释观众已经看到的内容。

## 十一、生产转换

将叙事场景拆成：

`场景节点 → 镜头单位 → 生成任务`

评估：人物一致性、动作、空间、道具、文字、声音、后期和备用方案。

每个关键机制至少准备：

- 理想实现；
- 稳定实现；
- 最低成本实现。
