# AI 原生视频架构师（中文版 Skill）

一套面向 **AI 短片、叙事视频、MV、广告概念片与实验影像** 的 Skill。

它不是简单地帮你“写提示词”或“加几个 AI 特效”，而是把主题、人物、情绪、故事或已有剧本，转化为一套由生成机制直接参与人物选择、剧情因果、世界规则、观众理解、主题表达和制作落地的 **AI 原生视频叙事系统**。

> 当前仓库仅保留中文版 Skill。显式调用名称：`$ai-native-video-architect-zh`

## 这个 Skill 能做什么

### 1. 从零创作 AI 原生短片

只给一个主题、人物处境、情绪或一句想法，Skill 会完成：

- 提炼真正的核心矛盾；
- 生成多个机制候选；
- 选择最适合当前类型的主机制；
- 建立世界规则；
- 设计人物、场景和结尾；
- 做 AI 原生性检查；
- 给出初步制作提醒。

示例：

```text
使用 $ai-native-video-architect-zh，
帮我设计一个90秒现实主义短片：
一个人持续在工作群里回复，却越来越像从未参与过项目。
```

### 2. 把传统剧本改造成 AI 原生版本

不是简单加上梦境、镜子、粒子消散、时空扭曲、人物透明或 AI 风格蒙太奇，而是先诊断原故事中哪些内容难以被传统影像直接表达，再设计统一的生成机制，使其真正进入人物行动和剧情。

示例：

```text
使用 $ai-native-video-architect-zh，
保留人物关系和原结局，把下面这个校园短片改造成AI原生版本。
改造强度用中等，不要变成高强度奇幻。
```

### 3. 诊断创意是否真的 AI 原生

Skill 可以判断一个方案目前属于：

- AI 工具化；
- AI 增强；
- AI 驱动；
- AI 原生；
- 高成熟度 AI 原生。

并检查：

- 主题与机制是否真正耦合；
- 删除 AI 机制后故事是否仍完全成立；
- 世界规则是否完整；
- 机制是否参与因果；
- 是否只是随机奇观；
- 是否落入常见 AI 视觉套路；
- 是否发生类型漂移；
- 是否具备制作可行性。

示例：

```text
使用 $ai-native-video-architect-zh，
只诊断下面的创意，不要替我重写。
请告诉我它为什么看起来像AI特效拼贴，以及最优先应该改什么。
```

### 4. 拆成可制作的镜头和生成任务

Skill 会区分：

```text
场景节点 → 镜头单位 → 生成任务
```

并给出：

- 场景转镜头表；
- 每个镜头需要传达的信息；
- 人物、空间、动作、道具和文字风险；
- 理想实现；
- 稳定实现；
- 最低成本实现；
- 首尾帧方向；
- 声音设计；
- 失败备用镜头；
- 最终可行性判断。

示例：

```text
使用 $ai-native-video-architect-zh，
不要改变这个短片的核心机制和结尾，
把它拆成90秒可生成镜头，并提供高风险镜头的替代方案。
```

## 适合哪些项目

当前版本优先支持：

- 60—180 秒 AI 叙事短片；
- 现实主义概念短片；
- 规则恐怖、心理恐怖、民俗恐怖；
- 规则喜剧、职场喜剧、家庭喜剧；
- AI 原生 MV 概念；
- 产品概念片和品牌叙事；
- 已有剧本的 AI 原生化改造；
- 进入图生视频前的生产适配；
- 创意压力测试和回归测试。

首版不以以下任务为重点：

- 长篇剧集完整统筹；
- 事实核查型纪录片；
- 实时更新具体模型价格与参数；
- 单纯生成一条图像提示词；
- 普通影评或文案润色；
- 只要求视觉风格、没有叙事需求的图片生成。

## 什么是 AI 原生视频

这里的“AI 原生”不是指所有画面都必须由 AI 生成。

真正的判断标准是：

> 如果移除生成机制，作品的核心表达、剧情因果、人物选择或结尾是否会明显受损？

### 不是 AI 原生的常见情况

```text
普通故事
+
梦境滤镜
+
粒子特效
+
几个超现实转场
```

这些画面即使漂亮，也可能只是 AI 增强。

### 更接近 AI 原生的情况

```text
人物具体处境
×
不可见矛盾
×
生成机制
×
世界规则
×
人物选择
×
类型呈现
×
结尾兑现
```

例如，主题是“劳动者的完整表达无法进入生产系统”。普通表达可能只是拍摄工厂很吵。AI 原生机制可以是：

> 流水线运行时，工人关于疼痛、休息和安全的完整句子会被机器节奏切碎，只有产量、速度和损耗等生产词汇能够被完整听见。

这个机制会直接改变：

- 管理人员理解了什么；
- 工人如何表达；
- 冲突如何升级；
- 为什么最终必须停机；
- 作品真正讨论的问题。

## 四种工作模式

用户不需要记住模式名称，Skill 会根据自然语言自动判断。

### Create：从零创作

适合：

- “帮我想一个短片”；
- “根据这个主题做一个创意”；
- “给我三个不同方向”；
- “根据这首歌设计 MV”。

默认输出：

1. 一句话概念；
2. 核心表达；
3. AI 原生机制；
4. 类型和气质；
5. 主要人物；
6. 5—8 个场景节点；
7. 结尾；
8. 制作提醒。

### Transform：已有方案原生化

适合已有故事、剧本、分镜或传统短片概念。

| 强度 | 保留内容 | 调整内容 |
|---|---|---|
| 轻度 | 情节、人物、结尾、大部分场景 | 增加统一母题和低强度世界响应 |
| 中度 | 人物、核心冲突、主要结尾 | 调整部分场景、世界规则和高潮 |
| 深度 | 主题、人物关系、关键情绪 | 重建世界规则、情节和结尾 |

默认使用中度改造。用户明确要求“不改结局”时，不得擅自改结局。

### Diagnose：诊断与评分

适合：

- 判断是否 AI 原生；
- 定位机制问题；
- 做压力测试；
- 比较多个方案；
- 检查类型漂移和套路风险。

诊断可分为快速诊断、标准诊断和深度诊断。用户说“只评价，不要改写”时，Skill 不应输出完整替代剧本。

### Adapt：制作适配

适合：

- 拆镜头；
- 拆生成任务；
- 人物一致性优化；
- 空间连续性处理；
- 首尾帧设计；
- 提示词方向；
- 降低制作难度；
- 设计备用镜头。

制作降级优先改变实现方式，不应直接删除创意核心。

混合请求按以下顺序执行：

```text
Diagnose → Transform → Adapt
```

## 快速安装

仓库中的 Skill 目录为：

```text
ai-native-video-architect-zh/
```

### Windows 用户级安装

将整个目录复制到：

```text
%USERPROFILE%\.agents\skills\ai-native-video-architect-zh
```

或者进入 Skill 目录后运行：

```powershell
.\scripts\install_windows.ps1
```

### macOS / Linux 用户级安装

复制到：

```text
$HOME/.agents/skills/ai-native-video-architect-zh
```

或者运行：

```bash
bash scripts/install_unix.sh
```

### 项目级安装

放入项目根目录：

```text
<项目根目录>/.agents/skills/ai-native-video-architect-zh
```

### 安装后

1. 重新打开 您的 agent 会话；
2. 未识别时重启 agent；
3. 使用显式调用做第一次测试。

```text
使用 $ai-native-video-architect-zh，
帮我做一个90秒关于职场隐形劳动的现实主义AI原生短片。
```

详细安装说明：[`ai-native-video-architect-zh/INSTALL.md`](ai-native-video-architect-zh/INSTALL.md)

先给标准方案，不要直接写完整分镜。

## 输出深度

### Level 1：概念速写

适合快速发散，通常包含一句话概念、核心机制、主要变化和结尾。

### Level 2：标准方案

默认等级，通常包含核心表达、类型与气质、世界规则、人物、场景结构、结尾和制作提醒。

### Level 3：完整创作包

包含创作输入卡、核心命题、候选机制、机制筛选、世界规则卡、类型控制、双线叙事、场景图谱、剧本和初步生产适配。

### Level 4：生产包

包含分场剧本、镜头表、生成任务、首尾帧方向、参考资源需求、声音设计、风险表、三档制作方案和失败备用路径。

用户未指定时，默认使用 Level 2，避免每次输出整套理论。

## 完整使用示例

### 示例一：从零创作现实主义短片

```text
使用 $ai-native-video-architect-zh。
设计一个90秒现实主义短片。
人物是一名住家保姆，她一直被雇主称为“家里人”，
但在家里没有真正属于自己的空间。
AI介入要克制，不要恐怖化，结尾必须是她主动离开。
```

预期工作：

- 提炼“亲近语言”与“实际权利”之间的矛盾；
- 发散空间、物体、关系和制度层机制；
- 选择低强度主机制；
- 保留雇主与劳动关系的现实责任；
- 避免让主人公通过自我和解解决结构问题；
- 让离开成为人物现实行动。

### 示例二：改造已有恐怖故事

```text
使用 $ai-native-video-architect-zh。
下面是我的家庭守夜故事。
保留人物、房屋和结尾，不增加鬼怪正脸。
请先诊断，再用“规则冲突型”改造，最后给出90秒场景结构。
```

自动路由：

```text
Diagnose → Transform
```

如果继续要求镜头，则追加 Adapt。

### 示例三：只诊断，不重写

```text
使用 $ai-native-video-architect-zh。
只评价下面的创意，不要改写剧本。
请从去AI测试、机制替换测试、因果参与、类型漂移和制作风险五方面分析。
```

### 示例四：进入制作适配

```text
使用 $ai-native-video-architect-zh。
这个方案已经确定，不改变主机制和结尾。
请拆成场景、镜头和生成任务，
并为人物一致性、多人动作、空间连续性和中文文字提供备用方案。
```

## 支持的类型控制器

### 现实主义

重点控制现实责任、异常强度、类型漂移、人物现实行动、认知变化，以及结尾是否把结构问题个人化。

### 恐怖

支持：

- 规则误判型；
- 规则缺失型；
- 规则冲突型；
- 观察污染型；
- 正常规则重释型；
- 软规则侵蚀型。

恐怖控制器会区分创作者、人物和观众分别知道多少，以及哪些未知必须保留。

### 喜剧

支持：

- 漏洞反噬型；
- 身份误认型；
- 群体默契型；
- 规则转移型；
- 成功利用型；
- 权力反转型；
- 观众误判型；
- 新秩序型。

喜剧基础规则通常应在一句话内理解，并分别检查笑点升级、主题升级、伤害阈值和结尾多样性。

### 通用叙事

当类型不明确、混合或不属于以上三类时，加载通用控制器，重点维持人物目标、规则因果、类型一致性、可理解性和结尾兑现。

## 验证与测试

```bash
python scripts/validate_package.py
```

验证内容包括：

- 文件是否与 `manifest.txt` 一致；
- `SKILL.md` 是否唯一；
- 内部引用是否有效；
- 关键目录是否齐全；

```bash
python scripts/score_concept.py
```

可用于记录主题耦合、世界规则完整度、叙事参与度、变化与收束、媒介不可替代性和 AI 美学自觉。

### 触发测试

```text
evals/trigger-tests.csv
```

包含 Create、Transform、Diagnose、Adapt 的正向请求，以及不应调用本 Skill 的负向样例。

### 回归测试

```text
evals/regression-cases.md
```

用于防止后续修改导致：

- 所有现实主义都温和和解；
- 所有恐怖都变成规则反转；
- 所有喜剧都变成漏洞反噬；
- 高频 AI 意象重新集中出现；
- 输出越来越长且无法直接使用。

## 常见问题

### 1. 为什么安装后没有自动触发？

先使用显式名称测试：

```text
$ai-native-video-architect-zh
```

然后确认：

- 整个文件夹被复制，而不是只复制 `SKILL.md`；
- 目录层级没有多套一层；
- 已新建 agent 会话；
- 必要时已重启 agent；
- `validate_package.py` 校验通过。

### 2. 可以只安装 SKILL.md 吗？

不建议。`SKILL.md` 是路由入口，完整能力依赖模式文件、类型控制器、机制库、场景图谱、生产适配、模板和评估文件。

### 3. 为什么默认输出没有完整评分表？

评分主要用于内部筛选和 Diagnose 模式。Create 和 Transform 默认优先输出可直接使用的方案。需要完整评分时可说：

```text
请做深度诊断，并显示百分制评分和全部压力测试。
```

### 4. 能不能直接生成某个模型的完整提示词？

可以进入 Adapt 模式生成模型无关的镜头任务和提示词方向。要适配具体模型，建议额外提供模型名称、生成方式、单段最大时长、首尾帧能力、参考图条件、预算和后期能力。

### 5. AI 原生是否意味着必须纯 AI 制作？

不是。推荐的实际制作方式通常是：

```text
参考图生成
+
局部图生视频
+
声音生成
+
传统剪辑
+
界面与文字后期
+
必要时实拍或3D辅助
```

AI 原生性来自叙事机制，而不是工具纯度。

### 6. 可以用于长篇项目吗？

可以作为概念和规则设计工具，但当前版本主要针对短片。长篇项目还需要额外处理角色长期弧线、多集规则升级、世界设定一致性、信息揭示节奏、机制疲劳和生产资产管理。

### 7. 为什么不鼓励镜子、透明人物和时间循环？

这些元素不是禁止，而是使用成本更高。使用时需要说明它们为什么不可替代地连接到当前人物、主题、类型、因果和结尾。

## 使用边界

这个 Skill 不应：

- 把每个故事都写成高概念规则片；
- 把人物变成机制演示工具；
- 把恐怖理解成黑暗滤镜；
- 把喜剧理解成夸张表情；
- 把现实主义理解成灰色调；
- 用神秘力量替代现实责任；
- 为了 AI 原生而牺牲可理解性；
- 为了降低制作难度直接破坏核心命题；
- 在用户只要求诊断时擅自重写；
- 在用户要求保留结局时擅自更换结局。
